## 2.1 Partial Functions and Computability

### 2.1.1 Partial Functions
Partial Recurvice Function은 계산가능하다(= 코딩이 가능하다)
- Input x State of the machine -> result(새로운 State)
  - State: Program 실행중에 변수들이 갖고있는 값들의 모든집합, memory에 들어가 있는 bit들이 0이냐 1이냐

# 
- Expressions do not always have values
  - Error termination: 3/0
    - (수학적으로 Infinity, In JavaScript ==> Nan, 나머지 언어들은 crash 남)
    - 계산했을때 우리가 예상, 만족할만한 값을 주는것이 아니다. 그것이 사고건, 의도했던간에

  - Nontermination : Program이 중단이 안되는경우
   
  - ``` function f(x){if (x==0) return 0 else return x+f(x-2);} ```
    - Parameter가 짝수인경우 짝수의 합을 return 하지만 홀수나, 음수의경우, 무한대로 Stack Frame을 쌓다가 Program이 터진다.

   # 
- Program이 Partial Function을 정의한다
  - 함수의 정의
    -  동일한 인자를 넣으면 Output이 같아야한다.
    - 정의역에 속하는 인자는 모두 대응되는 Return 값이 있어야함
  - Partial Function은 2번째 조건을 만족하지 않아도 된다.
    - 즉 어떤 Input에 대해 Mapping 되는 값을 주지 않을수도 있다.

--- 
### 2.1.2 Computability (계산 가능성)
컴퓨터에서 계산을 할수 있냐 없냐? 를 물어보는것임   
Computer Program은 Partial Recursive Function을 만듬 (모든 정의역에 속한 인자에 반드시 Return 값을 주지 않는 함수)

- 
  - Input이 있고 Output이 있으면 알고리즘에 따라 계산을 하다보면 Output을 내놓는게 계산가능 하다라고 말함
  - 모호하면 안됨
  - 유한한것을 요구해야함
  - 아무효과도 나타내지 않는것이아닌 효과를 일으켜야함

   
- Partial Recurvice Function은 컴퓨터에서 실행가능하다. 이론적으로
  - 여기서 이론적이란 의미는 저위에 4개의 조건을 만족하는데 수행하는 시간이 우주의 나이만큼 걸린다면 쓸모가 없다는 소리

- Partial Recursive Function, Turing machine, Lambda calculus 는 모두 똑같은 말이다.
- 모든 프로그래밍 언어는 Turing Complete 하다.
    - 모든 Partial recursive function을 만들수 있다.
